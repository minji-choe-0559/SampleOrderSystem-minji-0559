# UI 구성 제안 (초안, 구현 전 논의용)

원본 PPT 11p("메인 메뉴 예시 UI")를 비롯한 13/15/17/19/21/23p의 예시 화면 스타일을 참고해 정리한
제안이다. PRD.md는 "화면 세부 레이아웃은 자유롭게 결정한다"고 명시하므로, 이 문서는 구속력 있는
스펙이 아니라 **논의를 위한 시안**이다. 확정되면 PRD.md 5장 하위 절이나 별도 `SCREEN_SPEC.md`로
옮겨 반영한다.

## 1. 공통 레이아웃 원칙

- 화면 상단에 `====...====` 구분선 + 현재 메뉴 이름 + (필요 시) 타임스탬프. PPT 19p 예시(`[4]
  모니터링  2026-04-16 09:32:15`)를 그대로 채택.
- 화면은 "구분선 → 데이터 영역 → 구분선 → 선택 프롬프트(`선택 > `)" 3단 구조로 통일한다.
- 상태값(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`/`REJECTED`, ProductionJob의 `QUEUED`/
  `IN_PROGRESS`/`COMPLETED`)은 항상 대괄호 뱃지 표기(`[CONFIRMED]`)로 통일한다. ANSI 색상은
  터미널 호환성 문제가 있어 **기본은 무채색 텍스트 뱃지**로 하고, 색상 적용은 선택 사항으로 남긴다
  (결정 필요 — 3장 참고).
- 뒤로가기는 항상 `[0] 뒤로`, 종료는 메인 메뉴에서만 `[0] 종료`로 번호를 통일한다.
- 목록이 길어질 경우 PPT 13p처럼 `...외 N종  [N] 다음페이지` 방식으로 페이지네이션한다(한 화면에
  표시할 행 수는 고정 상수로 관리, 예: 10행).

## 2. 화면별 제안

### 2.1 메인 메뉴 (PPT 11p 대응)

```
============================================================
  반도체 시료 생산주문관리 시스템
============================================================
 시스템 현황   (프로그램 시작 시 자동 정산 실행됨)

 등록 시료  N종      총 재고     N,NNN ea
 전체 주문  N건      생산큐      N건 대기

------------------------------------------------------------
 [1] 시료 관리          [2] 시료 주문
 [3] 주문 승인/거절     [4] 모니터링
 [5] 생산 라인 조회     [6] 출고 처리
 [0] 종료
------------------------------------------------------------
 선택 > _
```

- PPT 예시의 "시스템 현황" 요약 라인을 유지하되, 문구를 "프로그램 시작 시 자동 정산 실행됨"으로
  덧붙여 PRD 5.5.4의 Catch-up 정산이 이미 반영된 수치임을 사용자에게 알린다.

### 2.2 시료 관리 (PPT 13p 대응)

- 진입 시 하위 메뉴(`[1] 시료 등록 [2] 시료 목록 [3] 시료 검색 [0] 뒤로`)를 먼저 보여준다(PPT
  13p와 동일한 2단 진입 방식).
- 시료 목록/검색 결과 테이블 컬럼: `ID | 시료명 | 평균 생산시간 | 수율 | 현재 재고`.
- 시료 등록 폼은 입력값을 한 번에 나열한 뒤 `[Y] 등록 [N] 취소` 확인 단계를 거친다(PPT 15p 주문
  화면과 동일한 확인 패턴을 시료 등록에도 통일 적용).

### 2.3 시료 주문 (PPT 15p 대응)

```
[2] 시료 주문
 시료 ID  > _
 고객명   > _
 주문 수량 > _

 입력 내용 확인
 시료   <이름> (<ID>)
 고객   <고객명>
 수량   <수량> ea

 [Y] 예약 접수  [N] 취소
 선택 > _

 예약 접수 완료.
 주문번호  ORD-YYYYMMDD-NNNN
 현재 상태 [RESERVED]
```

- 재고 확인 문구는 PPT 예시에서 "승인 메뉴에서 진행"으로 안내하는데, 그대로 유지(예약 시점에는
  재고를 보여주지 않고 승인 화면에서만 판단).

### 2.4 주문 승인/거절 (PPT 17p 대응)

- `RESERVED` 목록 → 번호 선택 → 재고 확인 결과 표시 → 승인/거절 확인.
- PRD 5.4/5.5에 따라 승인 시 분기 문구를 명확히 구분해 보여준다.
  - 재고 충분: `재고 충분 → 즉시 [CONFIRMED] 처리합니다.`
  - 재고 부족: `재고 부족. 부족분 N ea → 실생산량 N ea(수율 반영). ProductionJob 등록 후
    [PRODUCING] 처리합니다. 예상 완료 HH:MM:SS`
- PPT 예시의 "부족분만 생산" 표기를 유지하되, "실생산량"이라는 PRD 용어를 그대로 노출해 화면
  문구와 문서 용어를 일치시킨다.

### 2.5 모니터링 (PPT 19p 대응)

- 상단: 상태별 주문 건수 표(`RESERVED`/`CONFIRMED`/`PRODUCING`/`RELEASE`만 집계, `REJECTED`는
  집계 제외 — PRD 5.6 그대로).
- 하단: 시료별 재고 테이블(`시료명 | 재고 | 상태(여유/부족/고갈) | 잔여율 바`). 임계치는 PRD 8장
  결정 보류 사항이므로, 화면 문구는 미리 만들되 **임계치 상수는 결정 후 채운다**.
- 조회 진입 시 PRD 5.5.4에 따라 정산을 먼저 실행한 뒤 화면을 그린다는 점을 화면 하단에 작게 표기
  (`(조회 전 생산 정산 반영됨)`).

### 2.6 생산 라인 조회 (PPT 21p 대응)

- "현재 처리 중" 1건(ProductionJob 단일 라인 기준) + "대기 중인 주문(FIFO)" 목록 2단 구성.
- PPT 예시의 진행률 바(`72%`)는 실 진행 시간을 알아야 계산 가능한데, PRD는 IN_PROGRESS만 판정하고
  퍼센트 진행률까지는 정의하지 않는다(범위 밖일 수 있음 — 3장 논의 필요). 우선 진행률 바 없이
  `상태 [IN_PROGRESS] / 예정 완료 HH:MM:SS`만 표기하는 축소판을 기본안으로 제안.
- 대기열 컬럼: `순서 | 주문번호 | 시료 | 주문량 | 부족분 | 실생산량 | 예정 완료`.

### 2.7 출고 처리 (PPT 23p 대응)

- `CONFIRMED` 목록 → 번호 선택 → 즉시 처리 → 결과 표시(`[CONFIRMED] → [RELEASE]`).
- PPT처럼 확인 단계(Y/N) 없이 번호 선택 즉시 처리할지, 다른 화면처럼 확인 단계를 넣을지는 3장에서
  결정 필요(PPT 예시는 확인 단계 없음).

## 3. 구현 전 결정이 필요한 지점 (제안 검토 요청)

1. **상태 뱃지 색상 여부** — 무채색 vs ANSI 색상(Windows 콘솔 VT100 활성화 필요). 콘솔 환경에 따라
   깨질 수 있어 기본은 무채색 제안.
2. **생산 라인 진행률(%) 표기 여부** — PRD에 없는 파생 정보라 넣으려면 계산식(경과시간/총생산시간)
   을 PRD에 추가 확정해야 함.
3. **출고 처리 확인 단계 유무** — 다른 화면과의 일관성(항상 Y/N 확인) vs PPT 원안(확인 없음) 중
   택일.
4. **페이지네이션 행 수 상수화** — 예: 10행/페이지로 고정할지 여부.

## 4. 다음 단계

이 문서는 저장만 하고 구현하지 않는다. 위 3장의 미결정 사항에 대한 답을 받은 뒤, 확정 내용을
PRD.md 5장 또는 신규 `SCREEN_SPEC.md`로 승격하고 PLAN.md의 해당 Phase에 반영한다.
